## 现状（当前是依据什么搜）
- 后端会员数据来源是 `/supplier/list`，search 里传哪些字段，就对哪些字段做模糊匹配（LIKE）。字段含义：
  - `supplier`：会员卡号/名称
  - `contacts`：联系人（你说的“姓氏/姓名”通常在这里）
  - `telephone`：手机号码
  - `phonenum`：联系电话
  - 实现位置：[SupplierController.getList](file:///Users/lv/Workspace/erp/jshERP-boot/src/main/java/com/jsh/erp/controller/SupplierController.java#L71-L82)
  - SQL 条件是逐个 AND 拼起来（传了多个字段就会同时要求都匹配）：[SupplierMapperEx.xml](file:///Users/lv/Workspace/erp/jshERP-boot/src/main/resources/mapper_xml/SupplierMapperEx.xml#L4-L36)

## 前台收银里的“会员刷卡”当前交互
- 目前是“输入后按回车”或“点一次查询按钮”才会请求，并不会边输边出结果：
  - 输入框：`@pressEnter="handleMemberSearch"`
  - 查询按钮：`@click="handleMemberSearch"`
  - 代码位置：[CashierDeskModal.vue](file:///Users/lv/Workspace/erp/jshERP-web/src/views/cashier/modules/CashierDeskModal.vue#L129-L147)

## 关键问题（为什么你会觉得不够好）
- 你希望一个输入框同时支持“卡号/手机号/姓氏(姓名)”检索并返回列表。
- 但现在收银页的实现把同一个 key 同时塞给 `supplier/contacts/telephone/phonenum`（见 [CashierDeskModal.vue:handleMemberSearch](file:///Users/lv/Workspace/erp/jshERP-web/src/views/cashier/modules/CashierDeskModal.vue#L376-L394)），而后端 SQL 是 AND 关系，导致 key 不为空时往往匹配不到（要求同一个关键字同时出现在卡号、联系人、手机号等多个字段里）。

## 你想要的能力：可以做到
- “输入姓氏返回列表”：可以。应当对 `contacts like '%姓%'` 查询即可。
- “即时返回”：可以。前端做输入防抖（debounce）后自动触发查询；或者用可搜索下拉（showSearch）做远程联想。
