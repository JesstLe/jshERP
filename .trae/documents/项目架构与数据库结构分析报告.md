为了实现您截图中的“前台收银”功能，我们需要从数据库、后端 API 到前端页面进行全方位的开发。这个功能本质上是一个**服务资源管理（房台/座席管理）+ POS 收银**的综合模块。

以下是详细的前后端开发计划：

### 1. 数据库设计 (Database Design)

我们需要新增一张表来存储“房间”或“座席”的信息，并可能需要扩展现有的单据表来关联座席。

*   **新增表：`jsh_seat` (座席/房间表)**
    *   `id`: 主键
    *   `name`: 名称 (如 "修脚 1", "沙发 1号")
    *   `store_id` / `depot_id`: 所属门店/仓库 ID (关联 `jsh_depot` 或 `jsh_organization`)
    *   `status`: 状态 (0: 空闲, 1: 使用中, 2: 预订)
    *   `capacity`: 容量 (如截图中的 "1")
    *   `current_count`: 当前人数 (如截图中的 "0")
    *   `sort`: 排序字段
    *   `tenant_id`: 租户 ID
    *   `delete_flag`: 删除标记

### 2. 后端开发 (Backend Development)

需要在 `jshERP-boot` 项目中添加相应的业务逻辑。

*   **实体层 (Entity)**:
    *   创建 `Seat` 类，对应 `jsh_seat` 表。
*   **持久层 (Mapper)**:
    *   创建 `SeatMapper` 接口和 `SeatMapper.xml`，实现增删改查及状态更新。
*   **业务层 (Service)**:
    *   创建 `SeatService`。
    *   核心方法：
        *   `getSeatList(storeId)`: 获取某门店的所有座席及状态。
        *   `checkIn(seatId)`: 开台/入座操作，更新状态为“使用中”。
        *   `checkOut(seatId)`: 结账/清台操作，更新状态为“空闲”。
*   **控制层 (Controller)**:
    *   创建 `SeatController`，暴露 RESTful 接口供前端调用。
    *   接口示例：`/seat/list`, `/seat/add`, `/seat/update`, `/seat/action/checkIn`。

### 3. 前端开发 (Frontend Development)

需要在 `jshERP-web` 项目中开发新的收银工作台页面。

*   **路由配置**:
    *   添加新路由 `/cashier/index`，指向新的收银组件。
*   **页面布局 (Layout)**:
    *   **顶部栏**: 门店选择下拉框 (调用 `depot` 或 `organization` 接口)。
    *   **主体区域**: 使用 Grid 布局展示座席卡片。
        *   **卡片组件**: 显示座席名称、状态 (0/1)，根据状态改变背景色 (绿色-空闲, 蓝色-使用中)。
    *   **底部操作栏**: 固定底部的功能按钮组 (顾客开卡、签单清账、消费退货等)。
*   **交互逻辑**:
    *   **点击座席**:
        *   若**空闲**: 弹出“开单/入座”对话框，选择服务项目/技师。
        *   若**使用中**: 弹出“订单详情”对话框，进行加单或结账操作。
    *   **结账逻辑**: 跳转或弹窗调用现有的“销售出库”接口 (`/bill/sale_out`)，自动填充座席关联的商品和服务。

### 4. 实施步骤建议

1.  **第一步**：先创建 `jsh_seat` 数据库表。
2.  **第二步**：生成后端的基础代码 (Entity/Mapper/Service/Controller)。
3.  **第三步**：在前端搭建页面框架，实现座席列表的展示。
4.  **第四步**：实现“开台”和“清台”的状态流转逻辑。

如果您确认这个方案，我们可以从**创建数据库表**开始第一步。
